# Open Source vs Proprietary Boundary

## GPL-3.0 (This Repository — Open Source)

Everything in this repository is licensed under GPL-3.0:

- **RTSG Framework**: All mathematical theory, axioms, definitions, equations, proofs
- **Wiki Content**: 348+ pages of research, papers, monographs, companion papers
- **NRTE Brain-Graph Schema**: Database design, SQL schemas, Physarum dynamics functions
- **Compute Engine**: Rust cdylib for p-adic arithmetic, adelic products, Gram matrices
- **Frontend**: React RTSG Explorer prototype
- **Deploy Configs**: Caddy, PM2, PostgreSQL configs
- **Books**: 15 book manuscripts (monograph, companions, bestseller series, memoir)
- **Python Tools**: Verification scripts, wiki push tools

## Proprietary (NOT in this repository)

The following are trade secrets and proprietary intellectual property:

- **Trained Pattern Weights**: The specific numerical parameters learned from document analysis
- **Classification Models**: Neural network weights trained to classify documents using the I-vector/NRTE framework
- **Team Composition Engine**: The algorithms that use compatibility tensors (K-matrix) to optimally compose teams from individual I-vector profiles
- **Document Intelligence Scoring**: Trained models that score documents across the 12 intelligence dimensions
- **SaaS API Endpoints**: The hosted inference endpoints at smarthub.my

### Why This Split?

The GPL-3.0 framework gives the world the mathematical tools to understand intelligence geometrically. Anyone can build on it, study it, improve it, and publish derivative works — as long as they also open-source their modifications.

The proprietary layer is the **trained instantiation** of the framework on real-world data. Just as you can open-source a neural network architecture (GPL) while keeping your trained weights proprietary, we open-source RTSG while keeping our trained pattern classifiers as trade secrets.

This is the year to build the moat: accumulate training data, refine the classification weights, and establish the SaaS offering. The open-source framework attracts researchers and developers; the proprietary engines generate revenue.

### Patent Strategy

The mathematical framework itself is not patentable (abstract ideas). But the specific methods of applying the NRTE brain-graph to document classification, team composition, and intelligence profiling may be patentable as applied inventions. Patent applications should focus on:

1. Method for decomposing natural language documents into prime spectra across 12 intelligence dimensions
2. System for computing compatibility tensors between human cognitive profiles
3. Method for optimizing team composition using Physarum-inspired network dynamics on compatibility graphs
